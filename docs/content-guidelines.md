# Content guidelines - Maisonbionat

## Rôle de ce document
Ce document définit les règles éditoriales de Maisonbionat.

Objectif :
garantir un contenu cohérent, utile, lisible, crédible et compatible avec la stratégie SEO du site.

Le contenu doit toujours servir en priorité :
- l'utilisateur
- la clarté
- la structure du site
- la compréhension des sujets
- la navigation intelligente

## Ton général
Le ton du site doit être :
- sobre
- clair
- crédible
- pédagogique
- chaleureux sans excès
- utile avant tout

Le site ne doit pas adopter un ton :
- commercial agressif
- sensationnaliste
- pseudo-scientifique
- anxiogène
- militant
- lifestyle creux
- blog générique

## Règles de langage
- vouvoiement exclusif
- phrases claires
- vocabulaire accessible
- pas de jargon non expliqué
- pas d'effet de manche inutile
- pas de tournures gonflées artificiellement
- pas de promesses exagérées

Favoriser :
- formulations concrètes
- mots simples
- phrases compréhensibles à la première lecture
- progression logique

## Ce que le contenu doit toujours faire
Chaque page doit :
- répondre à un besoin clair
- aider à comprendre un sujet précis
- orienter vers les bonnes priorités
- rester utile même sans clic supplémentaire
- faciliter la poursuite de navigation si nécessaire

Le visiteur ne doit pas avoir l'impression de lire du remplissage.

## Types de pages
Le site comporte principalement :
- homepage
- pages piliers
- pages enfants
- FAQ ou blocs de questions
- contenus éditoriaux liés aux silos

Chaque type de page doit rester cohérent avec sa fonction.

## Homepage
La homepage doit :
- rester plus dense et structurée qu'une simple page de marque
- contenir de vrais textes utiles
- orienter sans noyer
- être plus éditoriale qu'un hero + cartes
- éviter l'effet index d'articles

Le texte de homepage doit être :
- bref mais informatif
- hiérarchisé
- orienté utilité immédiate
- cohérent avec les grands silos

## Pages piliers
Une page pilier doit :
- introduire clairement le sujet
- définir les grands enjeux
- hiérarchiser les priorités
- orienter vers les pages enfants
- être utile en elle-même
- éviter de devenir un simple annuaire de liens

### Structure recommandée d'une page pilier
1. H1 clair
2. introduction utile
3. cadrage du sujet
4. grands axes ou grands problèmes
5. sections structurées avec H2
6. passerelles vers les pages enfants
7. FAQ si réellement pertinente
8. conclusion utile

## Pages enfants
Une page enfant doit traiter :
- un sujet précis
- un problème précis
- une situation précise
- une pièce précise
- un angle suffisamment clair pour justifier une URL dédiée

### Structure recommandée d'une page enfant
1. H1 précis
2. introduction directe
3. explication du problème ou du sujet
4. solutions, critères ou éléments à connaître
5. conseils pratiques ou erreurs à éviter
6. maillage interne utile
7. FAQ si elle apporte une vraie valeur
8. conclusion courte et utile

## Titres
### H1
- un seul H1 par page
- clair
- naturel
- aligné avec l'intention principale
- pas de H1 flou ou purement marketing

### H2
- structurent la progression logique
- doivent annoncer un vrai sous-sujet
- doivent être utiles au lecteur
- éviter les H2 vagues ou génériques

### H3
- servent à détailler un point réel
- ne doivent pas être multipliés inutilement
- ne doivent pas créer une structure artificielle

## Règles de titrage
Favoriser :
- formulations concrètes
- titres orientés compréhension
- titres lisibles hors contexte
- titres utiles pour le SEO et pour l'humain

Éviter :
- titres sensationnalistes
- titres creux
- titres trop publicitaires
- variantes artificielles de mots-clés
- heading stuffing

## Introductions
L'introduction doit :
- poser immédiatement le sujet
- faire comprendre l'intérêt de la page
- annoncer la promesse réelle du contenu
- rester concise

Éviter :
- les intros trop longues
- les accroches vagues
- les phrases creuses
- les détours inutiles

Une bonne introduction doit faire sentir :
- ce dont parle la page
- pourquoi le sujet est important
- ce que le lecteur va comprendre

## Paragraphes
Les paragraphes doivent être :
- courts à moyens
- bien séparés
- faciles à parcourir
- organisés autour d'une idée principale

Éviter :
- gros blocs massifs
- répétitions
- phrases inutilement longues
- densité excessive

## Listes
Les listes doivent être utilisées quand elles améliorent réellement la lisibilité.

Les utiliser pour :
- critères
- priorités
- erreurs à éviter
- comparaisons simples
- étapes
- exemples

Éviter :
- listes décoratives
- listes redondantes avec le texte
- listes trop longues sans structure

## Tableaux
Les tableaux sont utiles quand ils simplifient une comparaison.

Les utiliser pour :
- comparer des solutions
- mettre en regard avantages et limites
- synthétiser des critères
- aider à choisir

Éviter :
- tableaux inutiles
- tableaux trop larges ou trop complexes
- tableaux répétés par automatisme

## FAQ
La FAQ doit être ajoutée seulement si elle apporte une vraie valeur.

Une FAQ utile :
- répond à de vraies questions
- complète la page sans répéter le corps du texte
- reste visible dans la page
- utilise des formulations naturelles

Éviter :
- FAQ de remplissage
- questions artificielles
- répétition pure du contenu déjà dit
- réponses trop courtes ou trop pauvres

## Maillage interne éditorial
Le maillage interne doit être :
- naturel
- descriptif
- utile
- cohérent avec le sujet

Chaque lien doit avoir une raison claire :
- approfondir
- comparer
- faire remonter vers la page mère
- orienter vers une page soeur pertinente

Éviter :
- liens forcés
- ancres vagues
- répétition des mêmes liens dans la même zone
- empilement de liens sans hiérarchie

## Conclusion
La conclusion doit être :
- courte
- utile
- cohérente avec la promesse de la page

Elle peut :
- résumer l'essentiel
- rappeler la priorité principale
- orienter vers une page complémentaire
- ouvrir sur le bon arbitrage

Éviter :
- conclusion artificielle
- répétition mot pour mot de l'introduction
- fin vide ou purement SEO

## CTA éditoriaux
Les CTA éditoriaux doivent rester sobres.

Ils peuvent :
- inviter à approfondir un sujet
- renvoyer vers un silo
- proposer un guide complémentaire

Ils ne doivent pas :
- casser le ton éditorial
- ressembler à des injonctions marketing
- être trop nombreux
- être agressifs

## Niveau de détail
Le contenu doit être :
- suffisamment concret pour aider
- suffisamment synthétique pour rester lisible
- proportionné à l'intention de recherche

Éviter :
- survol superficiel
- sur-développement inutile
- répétitions sémantiques pour faire du volume

## Rapport au SEO
Le SEO ne doit jamais détériorer :
- le ton
- la lisibilité
- la structure
- l'intérêt réel de la page

Le mot-clé principal doit être :
- présent naturellement
- intégré dans une formulation fluide
- visible aux bons endroits sans lourdeur

Éviter :
- répétition forcée
- reformulations artificielles
- blocs conçus seulement pour insérer des variantes

## Rapport à la santé et au naturel
Le site parle d'habitat sain, mais ne doit pas adopter un ton médical.

Autorisé :
- parler de confort
- parler d'air intérieur
- parler d'humidité
- parler d'irritation potentielle liée à certains produits
- parler d'environnement plus sain pour la famille

Éviter :
- diagnostic
- promesse de guérison
- exagération
- vocabulaire pseudo-médical
- raccourcis douteux

## Formulation des conseils
Les conseils doivent être :
- pratiques
- nuancés
- réalistes
- applicables

Favoriser :
- priorités
- arbitrages
- erreurs à éviter
- repères simples
- critères de choix

Éviter :
- recettes miracles
- injonctions irréalistes
- ton culpabilisant
- absolus non justifiés

## Recherche en amont
Avant de rédiger une page informative, effectuer une recherche web dès que le sujet demande :
- des faits précis
- des chiffres
- des comparaisons
- des repères techniques
- des normes
- des informations récentes ou évolutives

Objectif :
- identifier les sources les plus fiables
- éviter les approximations
- améliorer la pertinence réelle du contenu
- renforcer la crédibilité du site

Privilégier :
- sources officielles
- organismes reconnus
- documentation technique sérieuse
- publications de référence
- sources récentes quand le sujet évolue

Éviter :
- reprendre des affirmations non vérifiées
- s'appuyer uniquement sur des contenus commerciaux faibles
- inventer des chiffres ou conclusions non sourcées

## Cohérence entre pages
Toutes les pages doivent donner l'impression d'appartenir au même site.

Cela implique :
- même niveau de langue
- même logique de structure
- même clarté de ton
- même exigence d'utilité
- même sobriété éditoriale

## Interdictions absolues
- pas de ton alarmiste
- pas de greenwashing
- pas de pseudo-science
- pas de remplissage SEO
- pas de FAQ automatique sans utilité
- pas de titres sensationnalistes
- pas de paragraphes massifs
- pas de promesses excessives
- pas de contenu flou ou interchangeable

## Règle finale
À chaque page, se demander :
"Ce contenu est-il réellement utile, lisible, crédible et cohérent avec Maisonbionat ?"

Si la réponse est non, revoir l'angle, la structure ou le niveau de détail.
